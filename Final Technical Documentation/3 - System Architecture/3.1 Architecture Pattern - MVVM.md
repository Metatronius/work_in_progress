The app uses the Model-View-ViewModel (MVVM) pattern, which is the recommended approach for modern Android apps. It separates the code into three layers:

● View (UI Layer) – Displays data and captures user actions. It never touches the database directly.

● ViewModel – Holds and manages UI data, and survives screen rotations. It exposes state with LiveData.

● Model (Data Layer) – Contains the Room database, DAO, and repository. This is the single source of truth for task data.
