- **Activities (UI Layer)** – The four screens: `MainActivity` (task list + search), `AddTask` (new task form), `EditTask` (modify task), `TaskDetail` (read-only view). They observe `LiveData` and update automatically when data changes.

  
- **TaskViewModel** – The bridge between the UI and data. Validates input (e.g., title must be 1–30 characters, priority must be 0–3) and runs database operations inside coroutine scopes.

  
- **TaskRepository** – Sits between the ViewModel and the DAO. The ViewModel never talks to the database directly it always goes through here. This ensures a clean separation of concerns.

  
- **TaskDao** – Declares all the SQL operations using Room annotations. Returns a `Flow<List<Task>>` for the live task list and suspending functions for other operations (insert, delete, update, etc.).

  
- **AppDatabase (Room)** – The Room-managed SQLite database. Implemented as a singleton so the whole app uses the same database instance, with thread-safe access via double-checked locking.
