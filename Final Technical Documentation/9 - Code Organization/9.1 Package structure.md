

The production codebase is organized cleanly into focused functional layers under a flat layout: 


- **`com.example.taskmanager`**: Contains the primary presentation layer activities (`MainActivity`, `AddTask`, `EditTask`, `TaskDetail`, `CalendarActivity`) and broadcast infrastructure engines (`ReminderReceiver`, `ReminderScheduler`).

- **`com.example.taskmanager.database`**: Centralizes the complete data persistence pipeline, pairing the Room configuration (`AppDatabase`), database rules (`TaskDao`), source-of-truth orchestration (`TaskRepository`), table structures (`Task`), and transactional transport parameters (`TaskParams`).

- **`com.example.taskmanager.util` / `.extensions`**: Contains explicit validation frameworks (`DataUtil`) and structural instantiation helpers (`getTaskViewModel`).
